📄 ЛАБОРАТОРНАЯ РАБОТА 1: Разработка мобильных приложений (Android)

Наименование ПО: Интегрированная среда разработки «Android Studio».
1. Введение и краткое описание

Настоящее руководство пользователя разработано согласно ГОСТ Р 59795–2021. Android Studio — это специализированная интегрированная среда разработки (IDE), предназначенная для проектирования, написания, тестирования и отладки мобильных приложений для любых устройств на базе операционной системы Android. Программа включает в себя редакторы исходного кода, инструменты автоматизации сборки и встроенный эмулятор мобильных платформ.
2. Запуск программы

Запуск среды разработки в операционной системе Ubuntu осуществляется двумя стандартными способами:

    Через графический интерфейс (GUI): Открыть меню приложений, в строке поиска ввести «Android Studio» и кликнуть по иконке программы.

    Через терминал (CLI): Открыть консоль (Ctrl + Alt + T) и выполнить команду:
    Bash

    android-studio

3. Руководство по использованию (Перечень функций и приемы работы)

    Создание проекта: Выбрать File -> New -> New Project, определить шаблон активности (Activity) и язык программирования.

    Написание кода и интерфейса: Использование встроенного текстового редактора для логики приложения и визуального конструктора для разметки элементов экрана в XML.

    Тестирование на эмуляторе: Нажать кнопку Run (зеленый треугольник) на верхней панели для автоматической сборки проекта и запуска приложения внутри изолированной среды виртуального мобильного устройства.

4. Запланированный программой выход

Для корректного завершения работы необходимо сохранить текущие изменения в проекте (File -> Save All). Далее закрыть активные окна эмулятора и выбрать в главном меню пункт File -> Exit либо нажать комбинацию клавиш Ctrl + Q.
📄 ЛАБОРАТОРНАЯ РАБОТА 2: Высоконагруженные веб-сервисы (Backend)

Наименование ПО: Инструмент тестирования API «Postman».
1. Введение и краткое описание

Настоящее руководство пользователя разработано согласно ГОСТ Р 59795–2021. Postman представляет собой кроссплатформенное приложение, предназначенное для тестирования программных интерфейсов, автоматизации HTTP-запросов и проверки корректности обмена данными между серверной и клиентской частями веб-сервисов.
2. Запуск программы

Запуск программного обеспечения в среде Ubuntu выполняется следующими способами:

    Через графическую оболочку: Найти «Postman» в меню установленных приложений операционной системы.

    Через терминал: Ввести команду вызова snap-пакета:
    Bash

    postman

3. Руководство по использованию (Перечень функций и приемы работы)

    Формирование HTTP-запросов: Нажать вкладку + (New Request), выбрать метод (GET, POST, PUT, DELETE) и ввести URL-адрес локального сервера (например, http://localhost:5432).

    Настройка параметров: Заполнение вкладок Headers для передачи заголовков авторизации и Body (в формате JSON) для отправки данных на сервер.

    Анализ ответов сервера: После нажатия кнопки Send пользователь анализирует полученный код состояния (например, 200 OK или 404 Not Found), время задержки и структуру ответа в окне результатов.

4. Запланированный программой выход

Перед выходом необходимо убедиться, что все созданные запросы сохранены в коллекции (Ctrl + S). Завершение работы программы производится выбором пункта меню File -> Exit или нажатием на крестик закрытия окна в панели ОС.
📄 ЛАБОРАТОРНАЯ РАБОТА 3: Кроссплатформенная мобильная разработка

Наименование ПО: Комплект средств разработки «Flutter SDK».
1. Введение и краткое описание

Настоящее руководство пользователя разработано согласно ГОСТ Р 59795–2021. Комплект средств разработки Flutter SDK представляет собой набор консольных утилит, компиляторов и библиотек, предназначенных для создания кроссплатформенных мобильных приложений для iOS и Android из единой кодовой базы на языке Dart.
2. Запуск программы

Flutter SDK является интерфейсом командной строки (CLI) и функционирует внутри системного терминала Ubuntu. Первичный запуск и проверка готовности среды к работе осуществляются вводом команды:
Bash

flutter doctor

3. Руководство по использованию (Перечень функций и приемы работы)

    Создание нового проекта: Выполнить в целевой директории команду flutter create название_проекта для генерации структуры папок и базового шаблона приложения.

    Запуск отладки программного кода: Перейти в папку проекта (cd название_проекта) и запустить компиляцию командой flutter run, выбрав подключенное устройство или эмулятор.

    Горячая перезагрузка (Hot Reload): При изменении кода на языке Dart ввести символ r в активном окне терминала для мгновенного обновления интерфейса приложения на эмуляторе без полной перезаборки проекта.

4. Запланированный программой выход

Прерывание текущего сеанса отладки, выполнения сценариев или работы компилятора в терминале осуществляется нажатием интерактивной клавиши q в консоли либо стандартной системной комбинацией клавиш Ctrl + C.
📄 ЛАБОРАТОРНАЯ РАБОТА 4: Разработка 3D игр

Наименование ПО: Инструмент распределенного управления версиями и ассетами «Git LFS».
1. Введение и краткое описание

Настоящее руководство пользователя разработано согласно ГОСТ Р 59795–2021. Инструмент Git LFS (Large File Storage) является специализированным расширением для системы контроля версий Git, предназначенным для эффективного управления, версионирования и хранения крупных игровых ассетов (3D-моделей, тяжелых текстур, анимаций и аудиофайлов) отдельно от основного репозитория исходного кода.
2. Запуск программы

Утилита интегрируется в стандартную командную строку Linux. Активация и запуск расширения внутри локального каталога игрового проекта осуществляются вводом команды:
Bash

git lfs install

3. Руководство по использованию (Перечень функций и приемы работы)

    Регистрация типов игровых ресурсов: Для назначения отслеживания крупных бинарных файлов (например, 3D моделей формата FBX) выполняется команда git lfs track "*.fbx", которая создает или обновляет конфигурационный файл .gitattributes.

    Фиксация изменений в проекте: Добавление и фиксация измененных текстур и моделей происходят через стандартные приемы работы с Git: git add . и git commit -m "Добавлены 3D модели".

    Синхронизация с удаленным сервером: При выполнении команды git push Git LFS автоматически перехватывает тяжелые ассеты и загружает их на выделенный сервер хранения, заменяя их в основном репозитории текстовыми указателями.

4. Запланированный программой выход

Программа работает в транзитном (покомандном) режиме выполнения сценариев. Завершение работы утилиты происходит автоматически сразу после окончания выполнения вызванной команды Git; дополнительных действий пользователя для выхода не требуется.
📄 ЛАБОРАТОРНАЯ РАБОТА 5: Инфраструктура DevOps (Сборка и развертывание)

Наименование ПО: Сервер непрерывной интеграции и доставки «Jenkins».
1. Введение и краткое описание

Настоящее руководство пользователя разработано согласно ГОСТ Р 59795–2021. Jenkins — это расширяемая программная платформа для автоматизации процессов сборки, тестирования, контейнеризации и непрерывного развертывания (CI/CD) приложений на основе конфигурационных файлов и сценариев автоматизации.
2. Запуск программы

Jenkins работает как системная служба Linux Ubuntu. Управление службой и запуск осуществляются через терминал:
Bash

sudo systemctl start jenkins

После запуска графический веб-интерфейс программы становится доступен в браузере по локальному адресу: http://localhost:8080.
3. Руководство по использованию (Перечень функций и приемы работы)

    Создание сборочной задачи (Pipeline): В веб-интерфейсе выбрать Создать элемент, ввести имя и указать тип проекта (например, Pipeline).

    Конфигурирование автоматизации: Прописать сценарий сборки (Jenkinsfile), определяющий этапы получения кода из Git, тестирования, сборки Docker-контейнеров и мониторинга состояния.

    Запуск сборки: Нажать кнопку Собрать сейчас (Build Now) в меню проекта для инициации автоматического конвейера и отслеживания логов сборки в реальном времени.

4. Запланированный программой выход

Для корректного выключения сервера без прерывания текущих важных бизнес-процессов необходимо перейти в браузере по адресу http://localhost:8080/safeExit (безопасный останов после завершения активных задач) либо полностью остановить фоновую службу через терминал Ubuntu командой:
Bash

sudo systemctl stop jenkins

📄 ЛАБОРАТОРНАЯ РАБОТА 6: Кроссплатформенные настольные приложения

Наименование ПО: Инструмент сборки инсталляторов «Electron-builder».
1. Введение и краткое описание

Настоящее руководство пользователя разработано согласно ГОСТ Р 59795–2021. Electron-builder — это инструмент автоматизации сборки и упаковки кроссплатформенных настольных приложений, созданных с использованием веб-технологий (Node.js, JavaScript/TypeScript), в готовые нативные установочные пакеты (дистрибутивы формата .deb для Linux и инсталляторы под другие ОС).
2. Запуск программы

Запуск утилиты автоматизации сборки осуществляется из корневого каталога разрабатываемого десктопного проекта через консольный терминал Ubuntu. Инициация процесса происходит с помощью менеджера пакетов Node.js:
Bash

npm run dist

Или посредством прямого вызова установленного пакета: npx electron-builder --linux deb.
3. Руководство по использованию (Перечень функций и приемы работы)

    Упаковка ресурсов приложения: Программа автоматически компилирует исходные скрипты проекта, собирает веб-ресурсы и упаковывает их в изолированную среду выполнения.

    Создание установочных пакетов: Electron-builder автоматически генерирует исполняемый файл приложения и компилирует нативный установщик (.deb пакет), размещая его в создаваемой папке dist/.

    Первичная настройка сборщика: Конфигурирование параметров приложения (иконки, имя, права доступа и категории меню) осуществляется в текстовом формате внутри файла package.json в секции "build".

4. Запланированный программой выход

Утилита работает в автоматическом пакетном режиме. Программа самостоятельно завершает свою сессию работы и осуществляет выход сразу после успешного создания файлов инсталлятора, выводя отчет в консоль. При необходимости принудительного экстренного прерывания процесса сборки используется комбинация клавиш Ctrl + C.
